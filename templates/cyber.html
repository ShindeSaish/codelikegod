CheatCodes

• XSS

Type Description Example
Stored XSS Malicious scripts are permanently stored on a server (e.g., in a database or comment section). <script>alert('XSS')</script>
Reflected XSS Malicious scripts are reflected off a web server, usually via a URL or input field. https://example.com?q=<script>alert('XSS')</script>
DOM-Based XSS Malicious scripts manipulate the Document Object Model (DOM) directly on the client-side. document.write(location.hash) if the URL hash contains script tags.

• SQLi (SQL Injection)

Type Description Example
Classic SQLi Inject SQL code through input fields. SELECT * FROM users WHERE id='1' OR '1'='1';
Blind SQLi Extract data without direct feedback. AND (SELECT ASCII(SUBSTRING((SELECT DATABASE()),1,1))) > 64
Error-Based SQLi Exploit database error messages to gather information. ' UNION SELECT null, version(), null --

• Cross Site Request Forgery (CSRF)

Impact Description Example
Account Hijacking Forces a victim's browser to perform unintended actions. <img src="http://example.com/change-password?new=p@ss">
Data Modification Changes data using authenticated sessions. Submitting a POST request to update a profile.

• Parameter Tampering

Type Description Example
Price Manipulation Alter parameters to reduce item prices. item_price=10 → item_price=1
Privilege Escalation Modify user roles via parameters. role=user → role=admin

• Denial of Service (DoS)

Type Description Example
Network Flooding Overload network resources to disrupt services. Saturating a server with ping requests.
Application Crash Exploit application vulnerabilities to crash services. Sending malformed packets to a server.

• Distributed Denial of Service (DDoS)

Type Description Example
Botnet Attack Use compromised devices to attack a single target. Sending simultaneous requests from thousands of devices.
Amplification Exploit vulnerable services to amplify attack traffic. DNS reflection or NTP amplification.

• Man-in-the-Middle (MITM)

Impact Description Example
Data Interception Intercept and read sensitive information in transit. Capturing HTTP packets using Wireshark.
Session Hijacking Stealing session cookies to impersonate users. Using captured cookies to access a victim's account.

• Password Attacks

Type Description Example
Brute Force Attempts to guess passwords by systematically trying all possible combinations. Using a script to try passwords like `password1`, `password2`, etc.
Dictionary Attack Uses a predefined list of likely passwords to attempt access. Common list: `admin`, `123456`, `qwerty`.

• Zero-Day Exploit

Impact Description Example
Application Vulnerabilities Exploits an unpatched vulnerability in software. Attacking a newly discovered flaw in a web browser.
Critical Systems Targets vulnerabilities in critical infrastructure before a fix is available. Exploiting SCADA systems in industrial facilities.

Session Hijacking

Type Description Example
Cookie Stealing Stealing session cookies to impersonate a legitimate user. Using XSS to capture cookies and gaining access to a user's account.
Session Fixation Forcing a user to use a known session ID that can later be hijacked. Setting a session ID via a URL or a cookie, then using it to hijack the session.
Man-in-the-Middle (MITM) Intercepting session tokens during transmission. Intercepting cookies or session tokens from an insecure connection.
